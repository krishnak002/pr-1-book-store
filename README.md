# 📚 Book Store API

## Description

Book Store API is a RESTful backend application developed using **Node.js**, **Express.js**, and **MongoDB**. It provides a secure and efficient way to manage books and users through a set of API endpoints. The application supports user authentication, book management operations, request validation, and centralized error handling. Designed with a modular architecture, it is easy to maintain, scale, and extend with additional features in the future.

---

## ✨ Features

### 👤 User Management

* User Registration
* User Login
* Email-Based User Verification
* Duplicate User Prevention

### 📚 Book Management

* Add New Books
* View All Available Books
* Search Books by Title
* Update Existing Book Details
* Delete Books from Database

### ⚙️ Backend Features

* RESTful API Architecture
* MongoDB Database Integration with Mongoose
* Middleware-Based Request Validation
* Centralized Error Handling
* Environment Variable Configuration
* JSON Data Handling
* Clean and Modular Folder Structure
* Easy Integration with Frontend Applications
* Scalable and Maintainable Codebase

---

## ✅ Validations

### User Validation

* Name is required
* Email is required
* Password is required
* Password must contain at least 8 characters
* Existing users cannot register with the same email

### Book Validation

* Title is required
* Author is required
* Category is required
* Price is required
* Price cannot be negative

### Error Handling

* Invalid user input handling
* Missing field validation
* Duplicate user detection
* Book not found handling
* Global server error handling middleware
* Meaningful API response messages
